# Communication Guide for External Agents

이 문서는 외부 프로젝트의 에이전트(AI 및 개발자)가 `mdk_app_theme` 패키지를 활용하고 협력하는 방법을 정의합니다.

## 1. 패키지 활용 요약

- **목적**: MDK 제품군 통합 ThemeData 및 디자인 토큰 관리
- **핵심 진입점**: `package:mdk_app_theme/theme_utilities.dart`
- **주요 기능**: ThemeBrand 전환, 가변 폰트 제어, AdaptiveTheme 플랫폼 추상화
- **참조**: 자세한 설치 및 사용법은 [README.md](../README.md)를 확인하세요.

## 2. 고도화 및 수정 요청 프로세스

패키지에 새로운 기능이 필요하거나 버그 수정이 필요한 경우 다음 절차를 따릅니다.

1.  **Issue 생성 또는 공유**: Slack 채널 또는 GitHub Issue를 통해 의도를 먼저 전달합니다.
2.  **RFC (Request for Comments) 작성**: 복잡한 변경의 경우 아래 템플릿을 사용하여 `docs/EXTERNAL_REQUESTS.md`에 기록하거나 제안합니다.
3.  **검토 및 승인**: 본 프로젝트의 커뮤니케이터 에이전트가 내용을 검토하고 관리자 승인을 받습니다.

## 3. RFC / 수정 요청 템플릿

요청 시 아래 내용을 포함해 주세요.

```markdown
### [요청 제목]

- **요청자**: [에이전트 이름 / 프로젝트명]
- **유형**: [고도화 / 버그 수정 / 신규 브랜드 추가]
- **배경**: 왜 이 변경이 필요한가요?
- **상세 내용**: 구체적으로 어떤 기술적 변경을 원하나요?
- **영향 범위**: 기존 앱에 미치는 영향이 있나요?
```

## 4. 커뮤니케이션 원칙

- 모든 소통은 **한국어**를 기본으로 합니다.
- 변경 사항은 반드시 `CHANGELOG.md` 및 관련 문서 업데이트를 동반해야 합니다.
